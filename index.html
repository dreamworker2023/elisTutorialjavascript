L'esercizio "Calcoli sul Triangolo"

Ciao ragazzi!

Ho preparato questo manuale per analizzare insieme l'esercizio "Calcoli sul Triangolo". L'obiettivo è capire come i tre linguaggi fondamentali del web, HTML, CSS e JavaScript lavorano insieme per creare un'applicazione web interattiva.

Useremo l'app che abbiamo costruito come esempio per vedere la teoria in pratica. L'app ha un flusso semplice:

  1. Schermata Iniziale: Ci saluta con un titolo e due scelte: "Calcola Area" o "Calcola Perimetro".
  2. Schermate di Calcolo: A seconda della scelta, ci mostra la schermata giusta (per l'area o per il perimetro) con i campi (input) necessari per inserire i numeri.
  3. Azione: Ogni schermata ha un pulsante "CALCOLA" per vedere il risultato e un pulsante "Torna Indietro" per ricominciare.

⭐ Demo Interattiva ⭐

Qui puoi provare l'applicazione che analizzeremo. Giocaci un po' per capire il flusso descritto sopra.

HTML: Lo Scheletro (index.html)

L'HTML (HyperText Markup Language) definisce la struttura della nostra pagina. È come lo scheletro di un edificio: dice quali elementi ci sono e in che ordine.

Nel nostro file index.html (quello della demo!), le parti più importanti sono:

<head>

Qui mettiamo le "informazioni di servizio". La più importante per noi è il link al file CSS: 

<link rel="stylesheet" href="style.css">

<body>

Qui c'è tutto il contenuto visibile. Invece di creare tre file HTML diversi, abbiamo usato un trucco da "Single Page Application" (SPA): abbiamo messo tutte e tre le schermate nello stesso file, dentro tre <div> principali:

  • Schermata Welcome (<div id="welcome">)
  • Schermata Area (<div id="area">)
  • Schermata Perimetro (<div id="perimetro">)

IDs vs. Classi (Importante!)

ID (id="..."): È un identificativo unico. Lo usiamo per selezionare un elemento specifico (es. id="welcome").

Classe (class="..."): È un'etichetta di gruppo. Puoi dare la stessa classe a più elementi. Nel nostro codice, entrambi i pulsanti "Torna Indietro" hanno la class="btn-torna-welcome". Questo ci permette di programmarli entrambi con una sola riga di JavaScript!

Il Link a JavaScript

Questa riga, messa alla fine del <body>, carica ed esegue il file script.js. Lo mettiamo alla fine per essere sicuri che tutto l'HTML (i pulsanti, gli input, ecc.) sia stato caricato prima che lo script provi a selezionarli. 

<script src="script.js"></script>

CSS: Lo Stile (style.css)

Il CSS (Cascading Style Sheets) definisce l'aspetto della nostra pagina. È come l'arredamento e la pittura del nostro edificio.

Gestione delle Schermate

Il concetto più importante è la gestione delle schermate. All'inizio, vogliamo mostrare solo la #welcome e nascondere le altre. Usiamo questa "regola magica": 

/* Nasconde le sezioni non attive */
#area, #perimetro {
    display: none; 
}

Sarà poi JavaScript a cambiare questa proprietà, dicendo display: block; alla sezione che ci interessa e display: none; alle altre.

Altri concetti chiave del CSS:

  • Selettori: Il CSS usa i selettori per "agganciare" gli elementi HTML.
    • #welcome: Seleziona l'elemento con id="welcome" (il cancelletto # indica un ID).
    • .nav-button: Seleziona tutti gli elementi con class="nav-button" (il punto . indica una classe).
    • :focus: Una pseudo-classe che seleziona un input quando l'utente ci sta cliccando dentro.
  • Centratura: Per centrare tutto, abbiamo usato un metodo moderno sul <body>: 
    body {
    display: grid; 
    place-items: center; 
}
  • Classi Dinamiche: Abbiamo definito stili per .risultato-successo (verde) e .risultato-errore (rosso). Sarà JavaScript ad aggiungere queste classi per cambiare il colore del testo a seconda del risultato.

JavaScript: Il Cervello (script.js)

Il JavaScript (JS) è il cervello dell'applicazione. Gestisce l'interattività, risponde ai click, esegue calcoli e manipola l'HTML e il CSS.

Il nostro file è diviso in tre parti logiche:

1. Selezione degli Elementi (DOM)

Per prima cosa, "presentiamo" i nostri elementi HTML al JavaScript usando document e li salviamo in variabili const.

Per selezionare un ID singolo (è velocissimo): 

const sezioneWelcome = document.getElementById("welcome");

Per selezionare tutti gli elementi con una certa classe: 

const btnsTornaWelcome = document.querySelectorAll(".btn-torna-welcome");

Questo restituisce una collezione (simile a un array) di tutti i pulsanti "indietro".

2. Funzioni Logiche (Le Azioni)

Una funzione è un blocco di codice che esegue un compito specifico.

mostraSezione(idSezioneToShow)

Questa funzione gestisce la navigazione. Prima nasconde tutte le sezioni, poi mostra solo quella con l'ID che le abbiamo passato (es. "area"). Se stiamo tornando a "welcome", pulisce anche i campi.

calcolaAreaTriangolo() e calcolaPerimetroTriangolo()

Queste funzioni seguono uno schema logico preciso:

  1. Reset: Puliscono il risultato precedente.
  2. Lettura Valori: Prendono il testo dagli input usando .value (es. inputBase.value).
  3. Conversione: I valori dagli input sono stringhe! Dobbiamo convertirli in numeri usando parseFloat().
  4. Validazione (Controllo Dati): Fondamentale! Non ci fidiamo dell'utente.
    • Controlliamo se il valore non è un numero con isNaN().
    • Controlliamo che i numeri siano positivi (es. base <= 0).
    • (Solo perimetro) Controlliamo la Disuguaglianza Triangolare: la somma di due lati deve essere sempre maggiore del terzo.
  5. Calcolo: Se la validazione è OK, eseguiamo la formula matematica (es. (base * altezza) / 2).
  6. Output: Mostriamo il risultato all'utente.
    • divRisultatoArea.textContent = ... per impostare il testo.
    • .toFixed(2) per arrotondare a 2 decimali.
    • divRisultatoArea.classList.add("risultato-successo") per colorare il testo di verde (o "risultato-errore" per il rosso).

3. Collegamento degli Eventi (Event Listeners)

Questa è la "colla". Diciamo a JS: "Quando succede un 'click' su questo elemento, esegui questa funzione."

Per un pulsante singolo (come "Calcola Area"): 

btnCalcolaArea.addEventListener("click", calcolaAreaTriangolo);

Per i pulsanti di navigazione, usiamo una funzione anonima per passare un argomento: 

btnVaiArea.addEventListener("click", function() {
    mostraSezione("area"); 
});

Per i pulsanti multipli (i "Torna Indietro") usiamo il ciclo .forEach(). È elegantissimo: 

btnsTornaWelcome.forEach(function(bottone) { 
    bottone.addEventListener("click", function() {
        mostraSezione("welcome");
    });
});

Questo ciclo applica la stessa logica a tutti i pulsanti "indietro" in 3 righe di codice.

Glossario dei Comandi Utilizzati

Ecco un riepilogo dei comandi e concetti chiave.

HTML

  • <div>: Contenitore generico.
  • <input type="number">: Campo per inserire numeri.
  • <label>: Etichetta per un input.
  • <button>: Pulsante.
  • <link>: Collega un file CSS (nel <head>).
  • <script>: Collega un file JS (nel <body>).
  • id="...": Identificativo unico.
  • class="...": Etichetta di gruppo.

CSS

  • display: none;: Rende invisibile un elemento.
  • display: block;: Rende visibile un elemento.
  • display: grid;: Attiva il layout a griglia.
  • place-items: center;: Centra il contenuto in una griglia.
  • # (selettore ID): Seleziona per ID (es. #welcome).
  • . (selettore Classe): Seleziona per classe (es. .btn-torna-welcome).
  • :focus: Pseudo-classe per un elemento attivo.

JavaScript

  • document.getElementById("id"): Seleziona l'elemento con quell'ID.
  • document.querySelectorAll(".classe"): Seleziona tutti gli elementi con quella classe.
  • const: Dichiara una variabile costante.
  • function() { ... }: Definisce una funzione.
  • .addEventListener("click", ...): Ascolta un evento (click) su un elemento.
  • .style.display = "none": Modifica il CSS per nascondere.
  • .value: Ottiene il valore da un input.
  • parseFloat(stringa): Converte una stringa in un numero.
  • isNaN(valore): Controlla se un valore non è un numero.
  • .textContent = "...": Imposta il testo di un elemento.
  • .classList.add("classe"): Aggiunge una classe CSS.
  • .forEach(function() { ... }): Esegue un ciclo su ogni elemento di una collezione.
  • .toFixed(2): Arrotonda un numero a 2 decimali.
